# Color and Materials

## Color philosophy

Use color sparingly.

The UI should mostly rely on:
- neutral surfaces
- text contrast
- grouping
- spacing
- one main accent color
- semantic colors only when needed

## Accent color

Use a single main accent color for:
- selected states
- primary action
- focused controls
- key navigation emphasis

Do not apply accent color to everything clickable.

## Semantic colors

Use semantic colors only for meaning:
- success
- warning
- error
- destructive state

Do not use them as decoration.

## Light and dark mode

The design must work in both light and dark modes.

Check that:
- hierarchy remains clear
- text contrast stays strong
- muted surfaces do not become muddy
- emphasis still reads correctly

## Materials

Use stronger material treatment mainly for:
- toolbar
- sidebar
- sheets
- popovers
- inspectors
- top-layer controls when necessary

Do not overuse:
- blur
- glass
- shadows
- borders
- stacked elevated cards
