# Design Principles

## Goal

A macOS interface should feel native, calm, precise, and efficient.

The design should communicate:
- what this screen is for
- what matters most
- what the user should do next
- what is secondary

## Primary principles

### 1. Native before trendy
Prefer familiar macOS structure and behaviors over fashionable but generic UI patterns.

### 2. Structure before decoration
Hierarchy should come first from:
- layout
- grouping
- spacing
- typography
- emphasis

Do not rely on decoration to create meaning.

### 3. Efficient but not crowded
macOS can handle more information density than mobile, but dense is not the same as cluttered.

### 4. Quiet elegance
Aim for:
- restraint
- consistency
- visual rhythm
- controlled emphasis
- minimal noise

### 5. Long-term usability
Design for repeated use. The UI should still feel good after months of daily interaction, not just in a static mockup.
