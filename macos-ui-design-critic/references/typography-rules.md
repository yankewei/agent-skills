# Typography Rules

## Font direction

Use the system UI font style and native macOS typography feel.

Avoid decorative fonts in core application UI.

## Hierarchy

Prefer a simple 4-level hierarchy:
- page title
- section title
- body text
- supporting text

## Requirements

- page title must clearly anchor the screen
- section titles must organize the content
- body text must be easy to read
- supporting text should be quieter, but still legible

Avoid:
- too many font sizes
- weak difference between heading and body
- excessive tiny helper text
- using bold alone as the only hierarchy tool

## Line length

- keep long-form text constrained
- avoid extra-wide paragraph lines
- optimize labels and metadata for scanning

## Writing style

Use text that is:
- direct
- concise
- concrete
- plain-language
- action-oriented

Prefer labels like:
- Save
- Export
- Add Folder
- Continue
- Reveal in Finder

Avoid vague labels like:
- Confirm
- Submit
- Process
- Do It
