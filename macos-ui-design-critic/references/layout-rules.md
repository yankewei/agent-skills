# Layout Rules

## Preferred structural patterns

### Productivity app
- top toolbar
- left sidebar
- main content area
- optional right inspector

### Editor app
- top toolbar
- central editing area
- secondary controls in sidebar or inspector

### Utility app
- single main panel
- grouped sections
- one clearly dominant action area
- secondary settings moved into sheets or popovers

## Region rules

### Toolbar
Use for:
- high-frequency actions
- search
- sort/filter
- view switching
- compact status items

Do not overload with:
- long text
- low-frequency controls
- many unrelated actions

### Sidebar
Use for:
- navigation
- categories
- scopes
- collections
- content switching

Do not turn it into a large form area.

### Main content
This is the most stable region.
It should hold the main task:
- reading
- editing
- browsing
- inspecting core content

### Inspector
Use only when contextual details or properties matter.
Do not add one just because there is space.

## Spacing system

Use a stable rhythm:
- 24–32pt between major sections
- 16–24pt between groups
- 8–12pt between related controls
- 4–8pt between tightly related metadata or labels

Avoid:
- random spacing
- uneven padding
- titles too close to edges
- crowded groups next to empty groups

## Width and alignment

- Align related controls consistently
- Constrain long descriptive text
- Avoid very wide paragraph blocks
- Use readable content widths
- Make columns feel intentional
