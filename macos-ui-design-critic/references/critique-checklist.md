# Critique Checklist

When reviewing a macOS UI, inspect the following:

## Native feel
- Does it feel like a Mac app instead of a web dashboard?
- Is the structure window-friendly?
- Is there a clear toolbar/sidebar/content model where appropriate?

## Visual hierarchy
- Is the main purpose obvious immediately?
- Is there one clear primary action?
- Are secondary actions correctly de-emphasized?

## Layout and spacing
- Are sections clearly grouped?
- Is spacing rhythmic and consistent?
- Does any area feel cramped or oddly empty?

## Typography
- Is there a clear heading/body/supporting hierarchy?
- Are labels concise and readable?
- Is any text too small, too long, or too dense?

## Color and emphasis
- Is color restrained?
- Is there one coherent accent strategy?
- Are semantic colors used only for meaning?
- Will the design survive dark mode?

## Component quality
- Is the toolbar overloaded?
- Is the sidebar doing the right job?
- Are forms clean and vertical?
- Are lists/tables scanable?
- Are buttons correctly prioritized?

## Visual noise
Call out:
- too many cards
- too many borders
- too many shadows
- too many colored tags
- too many same-level actions
- mixed icon systems
- no focal point

## Final verdict
Conclude whether the UI is:
- native and refined
- acceptable but generic
- functional but visually weak
- structurally flawed
